# Project Instructions

## Context
This is a personal cash flow management application (Fluxo de Caixa Pessoal).

## Coding Standards
- Follow clean code principles
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Architecture
- Separate business logic from UI
- Use consistent file organization
- Follow DRY (Don't Repeat Yourself) principle

## File Organization
- Organize by feature or module
- Keep configuration files at root level
- Use appropriate subdirectories for assets, tests, etc.

## Development Workflow
- Test changes before committing
- Always confirm with the user before merging brachs, to make sure he can test the changes
- Document significant changes
- Keep the codebase clean and maintainable
